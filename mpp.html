How it works

The payment happens in the background.
You just set the price.

1

Agent requests your data

An AI assistant — acting on behalf of a customer or business — sends an HTTP request to your endpoint asking for a quote, availability, pricing, or any service you offer.

2

Your server asks for payment

Your server responds with 402 Payment Required — the standard HTTP code for "pay first." A HitPay payment challenge is included. The agent handles this automatically.

3

Payment processed, access granted

First time: agent's operator pays once and a token is stored. Every request after that: agent uses the token — HitPay charges it silently. Money appears in your dashboard tagged source=mpp.

🔐

Cryptographically verified

Every payment produces an Ed25519-signed receipt. Your server verifies it locally — no round-trip to HitPay needed. Nonce-binding prevents replay attacks.

Built on open standards

MPP uses HTTP's 402 Payment Required — the same protocol as Stripe's MPP and other implementations. Any HTTP client that speaks MPP can pay your server. HitPay provides the local payment rails (PayNow, DuitNow, SEA e-wallets) that global processors don't offer.

For developers

Wrap your existing endpoint.
Three lines of code.

Server — protect any endpoint

// Install: npm install @hit-pay/mpp import { mppGate } from '@hit-pay/mpp'; app.post('/api/classes/book', mppGate({ amount: 3.00, currency: 'SGD', apiKey: process.env.HITPAY_API_KEY, }), (req, res) => { // Only runs after payment is verified res.json(confirmBooking(req.body)); });

Client — agent pays automatically

// Install: npm install @hit-pay/mpp-client import { MppClient } from '@hit-pay/mpp-client'; const client = new MppClient({ paymentToken: process.env.AGENT_PAYMENT_TOKEN, }); // Handles 402s automatically — agent just // calls fetch(), MPP handles the rest const booking = await client.fetch('/api/classes/book', { method: 'POST', body: JSON.stringify({ class_id: 'hiit-0700-thu', employee_id: 'emp_8821' }) });
